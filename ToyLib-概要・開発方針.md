# 🎮 ToyLib 概要・開発方針

## ✅ コンセプト

- **専門学校生レベルをターゲット**にした、C++製の軽量ゲームエンジン（ライブラリ）
- **Actor/Componentベース**の構成で、UnityやUEをイメージしやすい設計
- 「**2〜3日で卒業制作レベルのゲームが作れる**」を目指す
- **入力・描画・移動・アニメ・衝突**といった基本要素が揃っており、学習用にも最適
- **デバッグ支援機能（Wireframe、BoundingVolume）**があり、内部挙動の可視化が可能
- STLやスマートポインタなど、**中級者向けC++のベース設計が学べる**

---

## 🧱 現在の開発方針

- **安全性と拡張性を重視**（例：Componentは`unique_ptr`で管理）
- **統一的なActor生成 (`CreateActor`) を通じた設計**
- **MoveComponentの派生クラスによる柔軟な動き（追従・公転など）**
- **Rendererクラスを中心とした描画制御（VisualLayerやShader管理）**
- **シャドウマッピングやスキンアニメなど基本的な3D機能の導入**
- **Jsonによる設定ファイル＆Factoryパターンによる自動生成予定**

---

## 🌱 拡張版に向けた展望（モダン化・今風技術のキャッチアップ）

### 🧩 UI・ツールまわり
- [ ] **ImGui統合**：リアルタイムインスペクタ、Transform編集、アニメ切り替えなど

### 🔄 アーキテクチャ面
- [ ] **ECS風構造の部分導入**：高速なデータ駆動アーキテクチャへ挑戦
- [ ] **非同期リソース読み込み**：マルチスレッドによるロード最適化

### 🚀 C++モダン化
- [ ] **C++20以降の機能活用**：concepts / ranges / coroutines / optional / variant など
- [ ] **テンプレート＆メタプログラミングの応用**：型安全性・汎用性の強化

### 📜 スクリプト連携（拡張案）
- [ ] LuaやAngelScriptを用いた軽量スクリプティング対応
- [ ] ノンプログラマー向けの挙動定義やイベント駆動の表現力を追加

4. **ECS的最適化や非同期対応による高速化**
